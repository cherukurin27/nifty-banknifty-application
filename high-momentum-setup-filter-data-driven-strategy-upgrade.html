Why does ADX 25–30 lose the most (−914 pts)? This is the "trap zone" — market just broke out of ranging (ADX crossed 20), Supertrend flips, RSI enters the buy band. But ADX 25–30 means the trend is early and fragile. It is also the zone where most false Supertrend flips occur (classic Type-C near-entry reversals). The STR trail is still narrow. One pullback = SL hit at tiny profit or loss.
BANKNIFTY shows the same pattern — even more extreme: ADX 20–25 (119 trades) = +4,389 pts. ADX 25–35 (98 trades) = −1,179 pts. The EOD winners don't care about "high ADX" — their avg ADX = 27.4. The SL losers avg ADX = 27.8. Essentially identical. ADX alone is not a momentum quality filter for this strategy.

The 5 Filters to Trade Only High-Momentum Setups

Filter 1: ST Gap Must Be <40 pts (NIFTY) / <90 pts (BNKN) IMPLEMENT

Rule: At entry, if abs(entry_price − st_value) >= 40 (NIFTY) or >= 90 (BNKN), skip the trade.

Why: NIFTY 40–50pt gap zone = 91 trades at 8% WR, −2,886 pts. BNKN 90–120pt gap = 70 trades at 13% WR, −4,640 pts. These trades have ST so far from price that the hard SL cap is immediately active — any retracement = full loss.

Filter 2: Block NIFTY RSI 45–52 Zone IMPLEMENT

Rule: For NIFTY, block all entries where RSI is in 45–52 (the no-man's land between SELL and BUY bands).

Why: RSI 45–48 = 13% WR, −312 pts on 15 trades. RSI 48–51 = 33% WR but only +18 pts (breakeven noise). This RSI zone generates signals where the market is neither trending down (clean SELL) nor surging up (clean BUY). It's the chop zone where Supertrend flips are most unreliable.

Already partially addressed: Current config has RSI_BUY_LOW = 53 and RSI_SELL_HIGH = 55, leaving a gap. The data confirms this gap is correctly excluded for BUY. But SELL signals at RSI 45–52 are still firing — and they lose.

Config change: lower RSI_SELL_HIGH = 55 → RSI_SELL_HIGH = 45 for NIFTY. Test impact on 90d + 365d.

Filter 3: BNKN RSI 40–50 Dead Zone Block TEST

Rule: For BANKNIFTY, block SELL entries where RSI is 40–50.

Why: RSI 40–45 = 20% WR, −1,562 pts (30 trades). RSI 45–50 = 21% WR, −1,331 pts (19 trades). Combined: 49 trades, −2,893 pts, 20.4% WR. These are SELL signals that fire when RSI is middling — not oversold enough for genuine selling pressure. The current SELL RSI cap of 55 allows these in.

Config: Add RSI_SELL_HIGH_BANKNIFTY = 40 (analogous to RSI_BUY_HIGH_BANKNIFTY). Or lower RSI_SELL_HIGH globally to 40 and test impact.

Filter 4: Weekly Trend Filter for BUY Direction TEST

Rule: Block BUY entries when the Nifty 50 weekly close is below its 10-week EMA. Allow SELL freely.

Why: NIFTY BUY at 365d = −101 pts on 72 trades (30.6% WR). Monthly data shows Aug 2025–Mar 2026 had sustained BUY losses: Nov −311 pts, Mar −244 pts, Jan −141 pts, Sep −196 pts. All bear-market months. A weekly EMA filter would have blocked most BUY entries during this entire 8-month stretch, preserving the SELL edge while avoiding counter-trend longs.

Requires fetching weekly candles for Nifty 50. Add WEEKLY_TREND_FILTER = True and WEEKLY_EMA_PERIOD = 10 to config. Compute on each trading day from weekly close data.

Filter 5: No New Entries When BNKN Entry is at ADX 25–30 AND RSI 40–50 Simultaneously TEST

Rule: Composite block — skip BNKN entries where both ADX is 25–30 AND RSI is 40–50.

Why: BNKN ADX 25–30 alone = −572 pts. BNKN RSI 40–50 alone = −2,893 pts. The intersection is where "early weak trend + neutral RSI" combine — signals that look valid on all 5 filters but have no real momentum conviction. This would be a hard gate without needing any new infrastructure.

Add to eval_entry_signal(): if symbol=="BANKNIFTY" and 25 <= adx <= 30 and 40 <= rsi <= 50: return SIGNAL_NONE.

Exact Config + Code Change for Filter 1 (ST Gap Max)

Step 1 — Add to config.py

# ST Gap filter: block entries where entry price is too far from ST line at moment of entry.
# NIFTY: gap >= 40 pts → 91 trades, 8% WR, -2,886 pts over 365d (near hard SL cap)
# BNKN:  gap >= 90 pts → 70 trades, 13% WR, -4,640 pts over 365d
ST_GAP_MAX = {
    "NIFTY"    : 40,   # pts — block if |entry - st_value| >= this
    "BANKNIFTY": 90,   # pts — block if |entry - st_value| >= this
}

Step 2 — Add to eval_entry_signal() in signal_engine.py (after ADX check)

    # ST Gap filter — block entries where ST band is too wide at entry
    st_gap_max = getattr(config, "ST_GAP_MAX", {}).get(symbol)
    if st_gap_max is not None:
        st_val_entry = float(row.get("st_value") or row.get("st") or 0)
        st_gap = abs(close - st_val_entry)
        if st_gap >= st_gap_max:
            return SIGNAL_NONE   # ST too far — near-cap entry, skip

3 Nifty BUY trade had a 3-point SL — ATR-based SL calculation anomaly SL Logic Bug

The Jul 17 BUY trade:

A 3-pt SL means the Supertrend value was 24,240.04 — i.e. the ST line was almost exactly at the close price. This happens when Supertrend has just flipped GREEN and the ST lower-band hasn't yet separated from price. The backtester code at backtester.py _open() does guard this: "if st_val >= close, fall back to hard-cap SL" — but the ST value was below close (24240.04 < 24243.00), so the guard didn't trigger.

The fix would be: enforce a minimum SL distance of e.g. max(sl_cap, close − 0.5 × ATR). A 3-pt SL on a 24,000-pt instrument will always be noise-hit. However this is a single edge case in the 10-day window — check if it recurs in 90-day data before tuning.

4 Monte Carlo 100% ruin probability is expected — not alarming here Expected / Cosmetic

P(ruin) is computed as: probability that MDD + net_pts < 0 across 1,000 shuffled simulations. With only 6 trades and net = +8 pts, any shuffle that starts with the 3 losers (−45, −45, −3) hits a drawdown of −93 pts — which exceeds the +8 net. So nearly every shuffle shows final equity < 0. The formula is working correctly; the stat is simply meaningless at N=6.

At 90 days you'd have ~60–80 trades and P(ruin) would drop to <5% (Nifty's historical profile). This is purely a sample-size artefact.

This is not a strategy flaw — it's a difficult weekly regime for SELL-biased strategies. The 90-day BNKN profile (not shown in this run) is the correct benchmark.
